데이터가 차곡차곡, 앱스 스크립트

입력받은 데이터를 어딘가에 저장해야 한다면 가장 친숙한 구글 스프레드시트가 제격입니다. 오늘은 학생들이 의견을 제출하면 내 시트에 실시간으로 기록되는 '의견 수합 앱'을 배포해 봅니다. 아래에서 작동 방식을 테스트해보세요.

완성 예시 · 학생 폼 화면과 교사 시트 화면
건의사항을 남겨주세요

시간 이름 의견
2026. 7. 13. 10:30 김철수 급식에 마라탕 원합니다.

앱스 스크립트 배포, 6단계

코드 에디터를 따로 설치할 필요 없이, 구글 시트 안에서 모든 작업이 이루어집니다. HTML 파일과 GS 파일을 나누어 넣는 것만 주의하면 됩니다.

구글 스프레드시트 준비하기

새 구글 스프레드시트를 열고, 첫 번째 행(A1, B1, C1)에 헤더를 적어줍니다. AI가 이 기준표를 보고 데이터를 넣게 됩니다.

A1: 시간
B1: 이름
C1: 의견

앱스 스크립트 열기

시트 상단 메뉴에서 확장 프로그램 → Apps Script를 클릭합니다.

새 창이 열리면 이것이 바로 우리의 '백엔드' 코드를 넣을 작업 공간입니다. 기본적으로 Code.gs라는 파일이 하나 열려 있을 거예요.

AI에게 역할 나누어 요청하기

앱스 스크립트는 기능(GS)과 화면(HTML) 코드를 분리해야 합니다. 아래 프롬프트를 복사해서 AI에게 명확히 지시하세요.

AI에게 보낼 요청 
구글 앱스 스크립트(GAS)를 이용해 학생들의 의견을 수합하는 웹 앱을 만들려고 해.
사용자가 폼에 이름과 의견을 적고 제출하면, 연결된 구글 시트(A열:시간, B열:이름, C열:의견)에 데이터가 저장되게 해줘.

요구사항:
1. 디자인은 학생들 모바일에서 보기 좋게 깔끔하게 꾸며줘. (CSS 포함)
2. 제출이 완료되면 알림창 대신 화면 글씨가 "제출 완료되었습니다!"로 바뀌게 해줘.
3. 내가 복사해서 붙여넣기 쉽도록, 앱스 스크립트용 백엔드 코드(Code.gs)와 프론트엔드 코드(Index.html)를 명확히 나누어서 전체 코드를 제공해줘.
팁 — Code.gs는 서버 역할을 하고, Index.html은 사용자에게 보여지는 폼 화면 역할을 합니다.

받은 코드를 각각 제자리에 붙여넣기

AI가 준 두 개의 코드를 앱스 스크립트 에디터에 알맞게 넣습니다.

1. Code.gs: 기본으로 열려있는 파일의 내용을 모두 지우고, AI가 준 Code.gs 코드를 붙여넣습니다.
2. Index.html: 에디터 왼쪽 위 파일(+) 버튼을 누르고 HTML을 선택합니다. 파일 이름을 Index로 짓고, 그 안에 AI가 준 HTML 코드를 붙여넣습니다.

코드를 모두 넣었다면 상단의 저장(💾 아이콘) 버튼을 꼭 눌러주세요.

웹 앱으로 배포하기

우측 상단의 파란색 배포 버튼을 누르고 새 배포를 선택합니다.

유형 선택(톱니바퀴)에서 웹 앱을 고른 후, 설정이 매우 중요합니다!

다음 사용자로서 실행: 나(내 이메일)
액세스하는 사용자: 모든 사용자 (이래야 구글 로그인을 안 한 학생도 폼을 냅니다)

설정을 맞췄다면 우측 하단의 배포 버튼을 누릅니다.

권한 허용 및 URL 공유 (가장 중요)

처음 배포할 때는 구글 시트에 접근할 권한 승인 창이 뜹니다. 화면이 무섭게 생겼지만 놀라지 마세요.

⚠ '안전하지 않은 앱' 경고창 돌파하기

1. 액세스 승인 버튼 클릭 → 내 구글 계정 선택

2. "Google에서 확인하지 않은 앱입니다" 경고창이 뜨면, 왼쪽 아래의 고급 글씨를 클릭합니다.

3. 맨 아래에 숨겨져 있는 (안전하지 않음) 제목 없는 프로젝트로 이동을 클릭하고 '허용'을 누릅니다.

※ 내가 짠 코드(AI가 짜준 코드)를 내 시트에 연결하는 것이므로 안심하고 넘어가도 됩니다.

권한을 허용하면 https://script.google.com/macros/s/.../exec로 끝나는 긴 주소가 나옵니다. 이 주소가 바로 우리 반의 **웹 앱 링크**입니다. 학생들에게 공유하세요!
