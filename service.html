Firebase 방명록, 7단계

앞의 두 유형과 달리 Firebase 콘솔에서 그릇(데이터베이스)을 먼저 만들고, 그 열쇠(설정)를 AI에게 건네 앱을 짓게 합니다. 순서대로 따라오세요. 한 반, 하루 행사 정도는 무료 등급(Spark)으로 충분해요.

Firebase 프로젝트 만들기

console.firebase.google.com에 구글 계정으로 접속해 프로젝트 만들기를 누릅니다. 이름을 짓고(예: our-class), 애널리틱스는 꺼도 됩니다. 이게 우리 앱의 ‘본부’예요.

Firestore 데이터베이스 만들기

왼쪽 메뉴에서 빌드 → Firestore Database → 데이터베이스 만들기를 누릅니다. 위치는 가까운 곳(asia-northeast3=서울)을 고르고, 시작 모드는 테스트 모드로 시작하세요.

여기가 방명록 글이 실제로 쌓이는 ‘창고’입니다.

⚠ 테스트 모드는 임시예요

테스트 모드는 30일 뒤 잠기고, 그 전엔 누구나 데이터를 읽고 지울 수 있어요. 6단계에서 보안 규칙을 꼭 설정합니다.

웹 앱 등록하고 ‘설정(config)’ 복사하기

프로젝트 개요 화면에서 웹 아이콘(</>)을 클릭 → 앱 별명을 짓고 앱 등록을 누릅니다. 그러면 firebaseConfig라는 코드 덩어리가 나와요. 이걸 통째로 복사해 둡니다. AI에게 건넬 열쇠예요.

이 열쇠, 노출돼도 되나요? — 네. config의 apiKey는 비밀번호가 아니라 ‘우리 프로젝트 주소’ 같은 거라, 화면 코드에 넣어 배포해도 됩니다. 대신 데이터를 지키는 일은 6단계 보안 규칙이 하니, 규칙 설정을 꼭 하세요.

AI에게 앱을 만들어 달라고 요청하기

이제 핵심이에요. 아래 프롬프트를 복사하고, 마지막 줄에 3단계에서 복사한 firebaseConfig를 붙여넣어 Claude(또는 쓰시는 생성형 AI)에게 보내세요.

AI에게 보낼 요청 ① 
Firebase Firestore를 이용한 '실시간 방명록' 웹 앱을 HTML 파일 하나로 만들어줘.

- 이름과 한 줄 메시지를 입력하고 '남기기' 버튼을 누르면 Firestore에 저장돼
- 저장된 메시지가 최신순으로 화면에 실시간으로 나타나게 해줘 (onSnapshot 사용)
- 각 메시지에 작성한 시간도 함께 보여줘
- 휴대폰에서 보기 좋게 반응형으로, 깔끔하게 꾸며줘
- Firebase 설정은 아래 값을 그대로 넣어줘:
  (여기에 3단계에서 복사한 firebaseConfig 붙여넣기)
- 전체를 index.html 하나에 담아, 복사해서 바로 쓸 수 있는 완성된 코드로 줘
팁 — 원하는 게 더 있으면 줄을 추가하세요. “욕설·비속어가 들어간 글은 저장 안 되게”, “글자 색은 파스텔톤으로”처럼요.

저장하고 확인하기

받은 코드를 index.html로 저장하고 더블클릭해 열어보세요. 방명록에 글을 남긴 뒤, Firebase 콘솔의 Firestore 화면을 새로고침하면 방금 쓴 글이 창고에 쌓여 있을 거예요. 데이터가 보이면 성공입니다.

보안 규칙도 AI에게 짜 달라고 하기 (가장 중요)

규칙 없이 두면 누군가 방명록을 통째로 지우거나 장난 글로 도배할 수 있어요. 규칙도 직접 짤 필요 없이, AI에게 시키면 됩니다.

AI에게 보낼 요청 ② 
방금 만든 방명록 앱의 Firestore 보안 규칙을 짜줘.

- 누구나 메시지를 읽고, 새 메시지를 쓸 수는 있게
- 단, 이미 올라온 메시지는 아무도 수정하거나 삭제하지 못하게
- 메시지 한 개의 길이는 200자로 제한

Firestore 규칙 코드와, 콘솔 어디에 붙여넣고 게시하는지도 알려줘.

받은 규칙 코드를 Firestore → 규칙 탭에 붙여넣고 게시를 누르면 적용됩니다.

인터넷에 올리기 (배포)

index.html이 든 폴더를 Netlify Drop에 끌어다 놓으면 링크가 생겨요. Firebase 콘솔의 Hosting에 올려도 됩니다. 자세한 방법은 메인의 ‘04 배포하기’를 참고하세요.

⚠ 공유 전 마지막 점검

학생들에게 링크를 뿌리기 전에 — 6단계 보안 규칙을 게시했는지 꼭 확인하세요. 규칙이 테스트 모드 그대로면, 누구나 방명록을 지울 수 있습니다.
